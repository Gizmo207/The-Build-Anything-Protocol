# Primary Users

## Developer (API Publisher)
- Wants to securely expose a database as an API without writing backend code.
- Wants to go from idea to working API in minutes.
- Wants to share an API without sharing database credentials.
- Wants confidence the API is safe against production data.
- Wants control over who can access the API and what they can do.

## Team Lead / Engineering Lead
- Wants a consistent way to publish internal APIs.
- Wants predictable governance and auditability.
- Wants to reduce bespoke backend work and maintenance burden.
